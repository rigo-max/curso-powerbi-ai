Módulo 01 · Fundamentos

Introducción a Power BI

Entiende el ecosistema completo de Power BI, sus herramientas y la nueva era de análisis potenciado por inteligencia artificial.

¿Qué es Power BI?

Power BI es la plataforma de Business Intelligence de Microsoft que permite conectar, transformar, modelar y visualizar datos de cualquier fuente. En 2024–2025, Microsoft integró Copilot (su IA generativa basada en GPT-4) directamente en Power BI, revolucionando la forma en que creamos reportes.

Herramienta gratuita para Windows. Aquí creas los modelos, medidas DAX y visualizaciones.
☁️
Power BI Service
Plataforma web (app.powerbi.com) para publicar, compartir y colaborar en reportes.
📱
Power BI Mobile
Apps iOS/Android para consumir dashboards desde cualquier lugar.
🤖
Power BI Copilot
IA generativa que crea reportes, escribe DAX y responde preguntas en lenguaje natural.

Power Query: El Editor M

Cada transformación en Power Query genera código M (Power Query Formula Language). Copilot puede escribir estas transformaciones en lenguaje natural.

Power Query M — Transformación de ventas
// Paso 1: Origen de datos
let
    Origen = Excel.Workbook(
        File.Contents("C:\datos\ventas2024.xlsx"), 
        null, true
    ),
    
    // Paso 2: Seleccionar hoja
    Ventas_Sheet = Origen{[Item="Ventas", Kind="Sheet"]}[Data],
    
    // Paso 3: Promover encabezados
    Encabezados = Table.PromoteHeaders(Ventas_Sheet, 
        [PromoteAllScalars=true]),
    
    // Paso 4: Filtrar nulls
    SinNulos = Table.SelectRows(Encabezados, 
        each [Monto] <> null),
    
    // Paso 5: Crear columna condicional
    ConCategoria = Table.AddColumn(SinNulos, 
        "Categoria", each 
        if [Monto] > 10000 then "Premium"
        else if [Monto] > 5000 then "Estándar"
        else "Básico")
in
    ConCategoria

Copilot en Power Query

🤖 Cómo usar Copilot para transformar datos

En el Editor de Power Query, haz clic en "Transformar datos con Copilot" y escribe en lenguaje natural: "Elimina filas donde la columna Ventas sea cero, convierte la columna Fecha a tipo Date y crea una columna nueva llamada Trimestre basada en el mes". Copilot generará el código M automáticamente.

Técnicas Esenciales de Transformación

Unpivot (Despivotar columnas)
Convierte columnas en filas. Esencial cuando tienes meses como columnas (Ene, Feb, Mar...) en lugar de una columna "Mes" con valores.
Merge Queries (Combinar tablas)
Equivalente al JOIN de SQL. Une tablas por una columna en común. Tipos: Inner, Left Outer, Right Outer, Full Outer, Anti.
Group By (Agrupar)
Agrupa filas y aplica funciones de agregación (Suma, Conteo, Promedio, Min, Max) generando una tabla resumida.
Custom Functions (Funciones personalizadas)
Crea funciones M reutilizables para aplicar la misma transformación a múltiples tablas o parámetros variables.
⚠️
IMPORTANTE: Query Folding

Cuando te conectas a bases de datos SQL, Power Query puede "delegar" las transformaciones al servidor. Mantén el Query Folding activo para maximizar el rendimiento. Puedes verificarlo clic derecho → "Ver pasos nativos de la consulta".

Módulo 03 · Modelo de Datos

Modelado de Datos

Un buen modelo es la base de todo. Aprende a construir modelos en estrella, dimensiones y tablas de hechos que potencian el rendimiento.

El Modelo en Estrella (Star Schema)

El modelo en estrella es el estándar de la industria para Power BI. Consiste en una tabla central de hechos (ventas, pedidos, transacciones) rodeada de tablas de dimensiones (clientes, productos, fechas, geografía).

📊
Tabla de Hechos (Fact Table)
Contiene las métricas numéricas (ventas, cantidades, costos) y las claves foráneas que la conectan con las dimensiones.
🗂️
Tabla de Dimensión (Dim Table)
Contiene atributos descriptivos: nombres, categorías, jerarquías. Son las tablas de "contexto" del análisis.
📅
Tabla de Fechas (DimDate)
La dimensión más importante. Debe tener un rango continuo de fechas y atributos como Año, Trimestre, Mes, Semana.

Creando la Tabla de Fechas con DAX

DAX — DimFecha completa
DimFecha = 
VAR FechaInicio = DATE(2020,1,1)
VAR FechaFin    = DATE(2026,12,31)
RETURN
ADDCOLUMNS(
    CALENDAR(FechaInicio, FechaFin),
    "Año",         YEAR([Date]),
    "Trimestre",    "Q" & QUARTER([Date]),
    "Mes Num",      MONTH([Date]),
    "Mes Nombre",   FORMAT([Date], "MMMM"),
    "Semana",       WEEKNUM([Date]),
    "Día Nombre",   FORMAT([Date], "dddd"),
    "Es Fin Semana", IF(WEEKDAY([Date],2)>5, TRUE(), FALSE()),
    "AñoMes",       FORMAT([Date], "YYYY-MM")
)

REGLA DE ORO

Las relaciones siempre deben fluir de las tablas de dimensiones hacia las tablas de hechos (de uno a muchos). Nunca crear relaciones bidireccionales a menos que sea absolutamente necesario, ya que degradan el rendimiento.

Módulo 04 · Cálculos

DAX: Lenguaje de Medidas

DAX (Data Analysis Expressions) es el corazón analítico de Power BI. Desde SUM hasta CALCULATE, aprende a dominar el lenguaje de fórmulas más potente del BI moderno.

Conceptos Fundamentales

📐
Medidas (Measures)
Cálculos dinámicos que se recalculan según el contexto del filtro. Son el tipo más común y recomendado.
📋
Columnas Calculadas
Se calculan fila por fila durante la carga. Ocupan memoria en el modelo. Úsalas solo para atributos estáticos.
🗃️
Tablas Calculadas
Tablas generadas completamente por DAX. Útiles para tablas de parámetros, fechas o resultados intermedios.

Funciones DAX Esenciales

CALCULATE — La función más poderosa

DAX — CALCULATE con múltiples contextos
// Ventas totales (básico)
Total Ventas = SUM(Ventas[Monto])

// Ventas solo de Lima
Ventas Lima = CALCULATE(
    [Total Ventas],
    DimCiudad[Ciudad] = "Lima"
)

// Ventas del año anterior (YoY)
Ventas Año Anterior = CALCULATE(
    [Total Ventas],
    SAMEPERIODLASTYEAR(DimFecha[Date])
)

// % Crecimiento
% Crecimiento = 
VAR Actual = [Total Ventas]
VAR Anterior = [Ventas Año Anterior]
RETURN
    DIVIDE(Actual - Anterior, Anterior, 0)

Copilot escribiendo DAX

🤖 Quick Measure Suggestions con IA

Desde el panel de datos, clic derecho → "Nueva medida rápida" → "Copilot". Escribe: "Calcula el porcentaje de ventas de cada categoría sobre el total, mostrando 0% si no hay datos". Copilot genera el DAX con DIVIDE, ALLSELECTED y manejo de errores automáticamente.

DAX avanzado — Ranking dinámico
// Top N clientes dinámico
Top Clientes = 
VAR RankActual = 
    RANKX(
        ALLSELECTED(DimCliente[Nombre]),
        [Total Ventas],
        ,
        DESC,
        DENSE
    )
VAR TopN = SELECTEDVALUE(Parametros[TopN], 10)
RETURN
    IF(RankActual <= TopN, [Total Ventas], BLANK())

Módulo 07 · IA Nativa

Smart Narratives & Q&A

Las funciones de IA nativas de Power BI (sin Copilot) que transforman datos en insights automáticos y permiten consultas en lenguaje natural.

Smart Narratives (Narrativas Inteligentes)

El visual de Narrativas Inteligentes genera automáticamente párrafos en lenguaje natural que describen los datos del reporte. Se actualiza en tiempo real cuando cambias filtros.

Activar el visual
Panel de visualizaciones → busca "Narrativa inteligente" (ícono con texto y gráfico). Agrega al canvas.
Personalizar el texto
Haz doble clic en el visual → "Agregar un insight de valor". Insertas dinámicamente medidas DAX dentro del texto narrativo: "Las ventas de [Mes] fueron [Total Ventas] representando [% Total]".
Integrar con filtros
El narrativo reacciona a todos los slicers y cross-filters del reporte. Cuando el usuario filtra por "Lima", el texto dice automáticamente "En Lima las ventas fueron..."

Q&A: Preguntas en Lenguaje Natural

Q&A permite a usuarios no técnicos hacer preguntas sobre los datos usando lenguaje coloquial. Power BI interpreta la pregunta y genera el visual adecuado automáticamente.

Ejemplos de preguntas Q&A en español
// Preguntas que Q&A entiende:

"ventas por mes en 2024 como gráfico de líneas"
"top 5 clientes por ingresos este año"
"mapa de ventas por departamento del Perú"
"cual es el producto más vendido en Lima"
"ventas totales comparadas con el año anterior"
"clientes con más de 3 compras en los últimos 6 meses"

Configurar sinónimos para Q&A

Q&A puede entender sinónimos en español. Configura en: Modelado → Q&A Setup → Synonyms. Por ejemplo: "ingresos" = "ventas" = "facturación" = "revenue".

Key Influencers Visual

Uno de los visuals de IA más potentes. Analiza estadísticamente qué factores tienen mayor impacto en una métrica objetivo usando regresión y correlación automática.

Configuración
Arrástara "Influenciadores clave" al canvas. En "Analizar" pon tu métrica (ej: Satisfacción). En "Explicar por" pon los factores (Región, Producto, Canal, Vendedor).
Interpretar resultados
El visual muestra: "Cuando el Canal es Digital, la Satisfacción es 2.3x mayor". Puedes cambiar entre "¿Qué influye en el aumento?" y "¿Qué influye en la disminución?".
Módulo 08 · ML Avanzado

Machine Learning con Azure

Integra modelos de Machine Learning reales en Power BI usando Azure ML y AutoML. Predicciones, clasificación y regresión directamente en tus reportes.

AutoML en Power BI Dataflows

Power BI Premium permite entrenar modelos de ML directamente desde Dataflows sin escribir código. Esto es AutoML (Automated Machine Learning) integrado.

Crear un Dataflow
En Power BI Service → Mi Workspace → Nuevo → Dataflow. Conecta tu fuente de datos históricos (ej: historial de ventas con columna "¿Compró de nuevo?" como target).
Aplicar modelos de ML
En el Dataflow → pestaña "Machine Learning" → "Agregar modelo ML". Selecciona: Predicción Binaria, Clasificación General, o Regresión.
Configurar el entrenamiento
Selecciona la columna objetivo (target), las columnas de entrada (features), y el período de entrenamiento. Power BI entrena el modelo automáticamente.
Aplicar predicciones
Una vez entrenado, aplica el modelo a datos nuevos. Power BI agrega columnas de predicción y probabilidad directamente en el Dataflow.

Integración con Azure Machine Learning

Power Query M — Llamada a Azure ML endpoint
let
    // Llamar al endpoint de Azure ML
    Endpoint = "https://[tu-workspace].eastus.inference.ml.azure.com/score",
    ApiKey   = "tu-api-key",
    
    // Construir el payload JSON
    Payload = Json.FromValue([
        data = Table.ToRecords(TablaPredicciones)
    ]),
    
    // Hacer la llamada REST
    Respuesta = Web.Contents(Endpoint, [
        Headers = [
            #"Authorization" = "Bearer " & ApiKey,
            #"Content-Type"  = "application/json"
        ],
        Content = Payload
    ]),
    
    // Parsear resultados
    Resultado = Json.Document(Respuesta)
in
    Resultado

Módulo 09 · NLP & Texto

Análisis de Sentimiento y NLP

Transforma texto no estructurado (reseñas, encuestas, tickets de soporte) en insights accionables usando IA de lenguaje natural integrada en Power BI.

Azure Cognitive Services en Power Query

Power BI tiene conectores nativos con Azure Cognitive Services para análisis de texto. Puedes procesar miles de registros de texto directamente en Power Query sin código Python ni R.

Power Query M — Análisis de Sentimiento
let
    // Importar función de Cognitive Services
    CognitiveFunctions = Diagnostics.Record(),
    
    // Función de sentimiento nativa (Power BI Premium)
    SentimientoTabla = Table.AddColumn(
        TablaResenas, 
        "Sentimiento", 
        each TextAnalytics.SentimentScore(
            [TextoResena],         // columna de texto
            "es",                // idioma español
            [ApiKey = MiApiKey]   // tu clave Azure
        )
    ),
    
    // Agregar detección de frases clave
    ConFraseClave = Table.AddColumn(
        SentimientoTabla,
        "FrasesClaves",
        each TextAnalytics.KeyPhrases([TextoResena], "es")
    )
in
    ConFraseClave

Módulo 10 · Detección IA

Anomaly Detection

Power BI detecta automáticamente anomalías en series de tiempo usando IA. Identifica picos, caídas y comportamientos inesperados sin configuración compleja.

Detección de Anomalías Nativa

En cualquier gráfico de líneas de Power BI, puedes activar la detección de anomalías con un solo clic. El algoritmo usa Isolation Forest + modelos estadísticos para identificar puntos fuera de lo normal.

Activar en gráfico de líneas
Selecciona el gráfico → Panel "Análisis" (ícono de lupa) → Busca "Buscar anomalías" → Actívalo. Los puntos anómalos aparecerán marcados automáticamente.
Ajustar sensibilidad
Desliza el control de "Sensibilidad" (1-100). Mayor sensibilidad = más puntos marcados. Ajusta según el nivel de ruido de tus datos.
Configurar formato
Personaliza el color del punto anómalo, el rango esperado (banda de confianza) y si mostrar el marcador con o sin banda de fondo.
Drill-down en anomalía
Clic en un punto anómalo → "Explicar por qué es anómalo". Power BI analiza las dimensiones relacionadas y muestra qué segmentos contribuyeron más.

Forecast (Pronóstico) en Gráficos

DAX — Apoyo al forecast con intervalos de confianza
// Límite superior del intervalo de confianza (manual)
Forecast Upper = 
VAR PromedioMovil = 
    AVERAGEX(
        DATESINPERIOD(DimFecha[Date], 
            LASTDATE(DimFecha[Date]), -30, DAY),
        [Total Ventas]
    )
VAR Desviacion = 
    STDEVX.P(
        DATESINPERIOD(DimFecha[Date],
            LASTDATE(DimFecha[Date]), -30, DAY),
        [Total Ventas]
    )
RETURN
    PromedioMovil + (1.96 * Desviacion)  // 95% IC
🎯 Casos de Uso: Anomaly Detection

Finanzas: detectar transacciones fuera de patrón (potencial fraude). Operaciones: alertar cuando una métrica de producción sale del rango esperado. E-commerce: identificar días de ventas inusualmente bajas o altas para investigar causa raíz. Clínicas: detectar picos en cancelaciones de citas.

Módulo 11 · Despliegue

Power BI Service & Publicación

Publica tus reportes, configura actualizaciones automáticas, gestiona permisos y comparte con stakeholders de forma segura.

Flujo de Publicación

Publicar desde Desktop
Archivo → Publicar → Selecciona workspace destino. El archivo .pbix se sube al Service incluyendo modelo de datos y reportes.
Configurar credenciales
En el Service → Dataset → Configuración → Credenciales del origen de datos. Ingresa las credenciales para que el Service pueda actualizar datos.
Programar actualización
Dataset → Configuración → Actualización programada. Configura frecuencia (hasta 8x/día en Pro, 48x/día en Premium). Activa notificaciones de fallo.
Gestionar permisos
Workspace → Acceso: Admin (control total), Miembro (publicar), Colaborador (editar), Visor (solo leer). Comparte reportes individuales con "Compartir enlace".
Crear App de Power BI
Agrupa múltiples reportes en una App para usuarios finales. Las Apps tienen su propia URL, navegación personalizada y branding corporativo.

Row Level Security (RLS)

RLS permite que distintos usuarios vean solo los datos que les corresponden, aunque compartan el mismo reporte. Un gerente de Lima ve solo datos de Lima; el de Arequipa, solo los suyos.

DAX — Filtro RLS dinámico por usuario
// Regla de seguridad en tabla DimVendedor
// Va en: Modelado → Administrar roles → Crear rol "Vendedor"

[Email] = USERPRINCIPALNAME()

// Para gerentes que ven su región completa:
[Region] = LOOKUPVALUE(
    DimGerentes[Region],
    DimGerentes[Email], USERPRINCIPALNAME()
)

Proyecto Final: Dashboard Ejecutivo con IA

Integra todo lo aprendido construyendo un dashboard ejecutivo real con Copilot, DAX avanzado, ML y narrativas automáticas para una empresa retail peruana.

📋 Descripción del Proyecto

Empresa: "RetailAndina SAC" — cadena de tiendas con 50 locales en Lima y provincias. Objetivo: Dashboard ejecutivo que muestre KPIs de ventas, predicción de demanda, análisis de sentimiento de reseñas de clientes y alertas de anomalías. Tecnologías: Power BI Desktop + Service + Copilot + Azure ML + Cognitive Services.

Arquitectura del Proyecto

Página 1: Executive Overview
KPIs principales (Ventas, Margen, Unidades, NPS). Gráfico de tendencia con anomaly detection. Smart Narrative con resumen automático. Filtros globales por Región y Período.
Página 2: Análisis de Ventas
Matrix de ventas por categoría y región con formato condicional. Gráfico de cascada P&L. Ranking de tiendas con conditional formatting. Forecast 90 días.
Página 3: Inteligencia de Clientes
Segmentación por clustering ML. Análisis de sentimiento de reseñas. Word cloud de temas mencionados. Key Influencers del NPS. Churn prediction score.
Página 4: Mapa Geográfico
Mapa de calor de ventas por distrito de Lima. Burbujas por tienda con ventas y margen. Drill-through a detalle de tienda individual.
Página 5: Q&A y Copilot
Visual Q&A para consultas libres. Panel Copilot activo. Botones de preguntas frecuentes pre-configuradas para usuarios no técnicos.

Medidas DAX del Proyecto

DAX — Medidas base del dashboard ejecutivo
// === TABLA DE MEDIDAS: _Metricas ===

// KPIs principales
Ventas Netas = SUMX(Ventas, Ventas[Cantidad] * Ventas[PrecioUnitario] * (1 - Ventas[Descuento]))

Margen Bruto = 
VAR VentasN = [Ventas Netas]
VAR Costo   = SUMX(Ventas, Ventas[Cantidad] * Ventas[CostoUnitario])
RETURN DIVIDE(VentasN - Costo, VentasN, 0)

// Crecimiento YoY con manejo de vacíos
Crecimiento YoY = 
VAR Actual   = [Ventas Netas]
VAR Anterior = CALCULATE([Ventas Netas], SAMEPERIODLASTYEAR(DimFecha[Date]))
RETURN
    IF(ISBLANK(Anterior), BLANK(), DIVIDE(Actual - Anterior, Anterior))

// Score de riesgo de churn por cliente
Churn Risk Score = 
VAR DiasUltimaCompra = DATEDIFF(
    CALCULATE(MAX(Ventas[Fecha]), ALLEXCEPT(DimCliente, DimCliente[ClienteID])),
    TODAY(), DAY())
VAR NumCompras = CALCULATE(COUNTROWS(Ventas), ALLEXCEPT(DimCliente, DimCliente[ClienteID]))
RETURN
    SWITCH(TRUE(),
        DiasUltimaCompra > 180, "Alto",
        DiasUltimaCompra > 90  && NumCompras < 3, "Medio",
        "Bajo"
    )
